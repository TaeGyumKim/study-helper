# 처음 사용자를 위한 설치 & 사용 가이드

컴퓨터를 잘 모르는 분도 이 문서만 따라하면 사용할 수 있습니다.

---

## 이 프로그램이 뭔가요?

숭실대학교 LMS 강의 영상을 **자동으로 시청 처리**(출석)하고, 원하면 **다운로드 + AI 요약**까지 해주는 프로그램입니다.

설정해두면 하루에 4번 자동으로 확인해서, 안 본 강의가 있으면 알아서 처리하고 텔레그램으로 요약을 보내줍니다.

---

## 준비물 체크리스트

시작하기 전에 아래 항목을 확인하세요.

- [ ] 숭실대 LMS 학번 + 비밀번호
- [ ] 컴퓨터에 Docker Desktop이 설치되어 있는가? (없으면 아래 1단계부터)
- [ ] (선택) 텔레그램 앱이 깔려 있는가?
- [ ] (선택) Gemini API 키가 있는가?

> 텔레그램과 Gemini API 키는 나중에 설정해도 됩니다. 일단 기본 기능(시청 처리)만 먼저 해볼 수 있어요.

## 1단계: Docker Desktop 설치

Docker는 이 프로그램을 실행하는 데 필요한 "가상 환경"입니다.

### Windows

1. https://www.docker.com/products/docker-desktop/ 에 접속
2. **"Download for Windows"** 클릭
3. 다운로드된 파일 실행 → 설치 완료까지 "Next" 클릭
4. 설치 중 **"Use WSL 2 instead of Hyper-V"** 체크박스가 보이면 **반드시 체크**
5. 설치 후 **컴퓨터 재시작**
6. 바탕화면의 **Docker Desktop** 아이콘을 더블 클릭해서 실행
7. 하단 작업표시줄에 고래 아이콘이 보이고, Docker Desktop 왼쪽 하단에 **초록색 "Engine running"** 이 뜨면 성공

> Docker Desktop을 처음 실행하면 약관 동의 화면이 나올 수 있습니다. "Accept"를 누르세요.

> **안 되는 경우?** → 아래 [문제 해결: Docker](#docker-관련-문제) 참고

### Mac

1. https://www.docker.com/products/docker-desktop/ 에 접속
2. **"Download for Mac"** 클릭
   - **Apple Silicon** (M1/M2/M3/M4 칩): "Apple Chip" 선택
   - **Intel** (2020년 이전 맥): "Intel Chip" 선택
   - 모르겠으면:  → 이 Mac에 관하여 → "칩" 항목 확인
3. `.dmg` 파일 열기 → Docker를 Applications 폴더로 드래그
4. Applications에서 Docker 실행
5. 상단 메뉴바에 고래 아이콘이 보이면 성공

## 2단계: 프로그램 파일 받기

아무 폴더에서나 시작할 수 있습니다. 바탕화면에 폴더를 하나 만들어봅시다.

### Windows (PowerShell)

1. 키보드에서 `Win + R` → `powershell` 입력 → Enter
2. 아래 명령어를 **한 줄씩** 복사해서 붙여넣고 Enter:

```powershell
cd ~/Desktop
mkdir study-helper
cd study-helper
```

3. 필요한 파일 2개를 다운로드합니다:

```powershell
curl -o docker-compose.yml https://raw.githubusercontent.com/igor0670/study-helper/main/docker-compose.yml
curl -o .env https://raw.githubusercontent.com/igor0670/study-helper/main/.env.example
```

4. 파일이 제대로 받아졌는지 확인:

```powershell
dir
```

`docker-compose.yml`과 `.env` 두 파일이 보여야 합니다.

### Mac (터미널)

1. Spotlight(`Cmd + Space`)에서 "터미널" 검색 → 실행
2. 아래 명령어를 **한 줄씩** 복사해서 붙여넣고 Enter:

```bash
cd ~/Desktop
mkdir study-helper
cd study-helper
curl -O https://raw.githubusercontent.com/igor0670/study-helper/main/docker-compose.yml
curl -o .env https://raw.githubusercontent.com/igor0670/study-helper/main/.env.example
```

3. 파일이 제대로 받아졌는지 확인:

```bash
ls
```

`docker-compose.yml`과 `.env` 두 파일이 보여야 합니다.

> **파일이 안 보이는 경우?** → 인터넷 연결을 확인하고 curl 명령어를 다시 실행하세요.

## 3단계: 프로그램 실행

### 최초 실행

**반드시 `study-helper` 폴더 안에서** 실행해야 합니다:

```bash
docker compose run --rm study-helper
```

> **주의**: `docker compose up`은 사용하지 마세요. 화면이 깨집니다. 반드시 `run --rm`을 사용하세요.

> 처음에는 프로그램 이미지를 다운로드하므로 3~5분 걸릴 수 있습니다. `Pulling...`, `Downloading...` 메시지가 나오면 정상입니다. 기다려주세요.

### 로그인

실행되면 아래 화면이 나타납니다:

```
+-----------------------------------------------------------------------------+
|                         숭실대학교 LMS Study Helper                         |
+-----------------------------------------------------------------------------+

  LMS 계정정보를 로그인하세요.

  학번: (여기에 학번 입력)
  비밀번호: (여기에 비밀번호 입력)
```

학번과 비밀번호를 입력하면 자동으로 암호화되어 저장됩니다. **다음부터는 자동 로그인**됩니다.

> 비밀번호 입력 시 화면에 아무것도 안 보이는 게 정상입니다. 그냥 입력하고 Enter를 누르세요.

> **로그인 실패?** → 아래 [문제 해결: 로그인](#로그인-관련-문제) 참고

### 과목 목록

로그인에 성공하면 과목 목록이 나타납니다:

```
  #    과목명                  미시청 / 전체    학기
 ─────────────────────────────────────────────────
  1    소프트웨어공학            3 / 12        2026-1
  2    데이터베이스              0 / 10        2026-1

  과목 선택 (0: 종료 / setting: 설정 / auto: 자동 모드):
```

| 입력 | 동작 |
|------|------|
| 숫자 (1, 2, ...) | 해당 과목의 강의 목록으로 이동 |
| `0` | 프로그램 종료 |
| `setting` | 설정 화면 이동 |
| `auto` | 자동 모드 시작 (아래 6단계 참고) |

## 4단계: 강의 시청 처리 (출석)

1. 과목 목록에서 과목 번호 입력
2. 주차별 강의 목록이 나타남 — 시청할 강의 번호 입력
3. 메뉴가 나타남:

```
  1. 재생
  2. 다운로드
  3. 취소
```

4. `1`을 입력하면 **백그라운드 재생**이 시작됩니다

> **"백그라운드 재생"이란?** 실제로 영상을 보는 게 아니라, LMS 서버에 "이 영상을 봤다"는 신호를 보내는 겁니다. 소리도 안 나고 화면도 안 보입니다. 강의 길이만큼의 시간이 지나면 자동으로 출석 처리됩니다.

5. 재생이 끝나면 자동으로 출석 처리 완료

> **재생 중 기다리기 싫다면?** `auto` 모드를 사용하세요 (6단계). 켜놓으면 알아서 전부 처리합니다.

## 5단계: 추가 기능 설정 (선택)

과목 목록 화면에서 `setting` 입력으로 설정 화면에 접근합니다.

### AI 요약 설정 (Gemini) — 무료

강의 내용을 자동으로 요약해줍니다.

> **Gemini API = Google API 입니다.** Gemini는 Google이 만든 AI 모델이고, Google AI Studio에서 발급받는 API 키를 사용합니다. "Google API 키"와 "Gemini API 키"는 **같은 것**입니다. 헷갈리지 마세요!

1. [Gemini API 키 발급 가이드](gemini-api-key.md)를 따라 API 키를 발급받습니다
   - 간단 요약: https://aistudio.google.com 접속 → 로그인 → "Get API key" → "Create API key" → 복사
2. 설정 화면에서 아래 항목을 설정:
   - AI 요약: **사용**
   - AI 에이전트: **gemini**
   - API 키: 발급받은 키 붙여넣기 (`AIzaSy...`로 시작하는 긴 문자열)

> Gemini API는 무료 티어가 있어서 개인 학습 용도로는 돈이 들지 않습니다.

### 텔레그램 알림 설정

처리 결과를 텔레그램으로 받을 수 있습니다. 자동 모드와 함께 사용하면 편리합니다.

1. [텔레그램 설정 가이드](telegram-setup.md)를 따라 봇을 생성합니다
2. 설정 화면에서 아래 항목을 설정:
   - 텔레그램 알림: **사용**
   - 봇 토큰: BotFather에서 발급받은 토큰
   - Chat ID: @userinfobot에서 확인한 숫자

### 다운로드 설정

| 옵션 | 의미 | 용량 |
|------|------|------|
| video | mp4 영상 파일만 저장 | 많음 (수백 MB) |
| audio | mp3 음성 파일만 저장 | 적음 (수십 MB) |
| both | mp4 + mp3 둘 다 저장 (기본값) | 가장 많음 |

다운로드된 파일은 `study-helper/data/downloads/과목명/N주차/` 폴더에 저장됩니다.

### .env 파일 직접 편집하기 (선택)

프로그램 안의 `setting` 화면 대신 `.env` 파일을 직접 열어서 설정할 수도 있습니다.
**프로그램이 실행되지 않는 상황**에서 설정을 바꾸고 싶을 때 유용합니다.

#### .env 파일 여는 법

| OS | 방법 |
|----|------|
| Windows | 탐색기에서 `study-helper` 폴더 열기 → `.env` 파일을 **메모장**으로 열기 |
| Mac | 터미널에서 `open -a TextEdit .env` 입력 |

> `.env` 파일이 안 보이면 **숨김 파일 표시**를 켜야 합니다.
> - Windows: 탐색기 → 보기 → "숨긴 항목" 체크
> - Mac: `Cmd + Shift + .`

#### .env 파일 설정 예시

아래는 **자동 모드까지 전부 사용**하는 완전체 설정 예시입니다.
`=` 뒤에 값을 채워넣으면 됩니다. 따옴표(`"`, `'`)는 **붙이지 마세요**.

```ini
# ── 계정 (프로그램 실행 시 입력하면 자동 저장됨) ─────────────
LMS_USER_ID=20241234
LMS_PASSWORD=mypassword123

# ── 다운로드 ────────────────────────────────────────────────
DOWNLOAD_DIR=              ← 비워두면 기본 폴더 사용 (Docker: /data/downloads)
DOWNLOAD_RULE=both         ← video / audio / both 중 택 1

# ── STT (음성→텍스트 변환) ──────────────────────────────────
STT_ENABLED=true           ← true 또는 false
WHISPER_MODEL=base         ← tiny / base / small / medium / large
STT_LANGUAGE=ko            ← ko(한국어) / en(영어) / 비우면 자동 감지

# ── AI 요약 ─────────────────────────────────────────────────
AI_ENABLED=true            ← true 또는 false
AI_AGENT=gemini            ← gemini 또는 openai
GEMINI_MODEL=gemini-2.5-flash
GOOGLE_API_KEY=AIzaSy...   ← Google AI Studio에서 발급받은 키 (= Gemini API 키)
OPENAI_API_KEY=            ← OpenAI 사용 시에만 입력
SUMMARY_PROMPT_EXTRA=      ← 추가 요약 지시사항 (비워두면 기본값)

# ── 텔레그램 알림 ───────────────────────────────────────────
TELEGRAM_ENABLED=true      ← true 또는 false
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHI...  ← BotFather에서 발급
TELEGRAM_CHAT_ID=123456789                  ← @userinfobot에서 확인
TELEGRAM_AUTO_DELETE=false ← true: 요약 전송 후 파일 자동 삭제
```

#### 자주 하는 실수

| 실수 | 올바른 예 |
|------|-----------|
| 값에 따옴표 붙이기 | `GOOGLE_API_KEY=AIzaSy...` (O) / `GOOGLE_API_KEY="AIzaSy..."` (X) |
| `=` 앞뒤에 공백 넣기 | `AI_ENABLED=true` (O) / `AI_ENABLED = true` (X) |
| 줄 끝에 주석 달기 | 줄 끝 `← 설명` 부분은 **실제 파일에 넣지 마세요** (위는 설명용) |
| 빈 줄 삭제 | 빈 줄이 있어도 상관없습니다 |

> `.env` 파일을 수정한 후에는 프로그램을 **재시작**해야 반영됩니다.

## 6단계: 자동 모드 (핵심 기능)

**한번 켜두면 하루종일 알아서 해줍니다.**

### 자동 모드 시작

1. 과목 목록에서 `auto` 입력
2. 스케줄 설정 — Enter만 누르면 기본값(09시, 13시, 18시, 23시) 사용
3. 즉시 실행 여부 선택 — `y`를 누르면 바로 시작

### 자동 모드가 하는 일

설정된 시간마다 자동으로:

```
1. LMS에서 안 본 강의 확인
2. 백그라운드 재생 (출석 처리)  ← 실패 시 최대 3회 재시도
3. 영상 다운로드
4. 음성 추출 (mp3)
5. 텍스트 변환 (STT)
6. AI 요약
7. 텔레그램으로 요약 전송
```

### 자동 모드 종료

`0`을 입력하고 Enter를 누르면 종료됩니다.

### 자동 모드 필수 조건

자동 모드를 쓰려면 아래 3가지가 **모두** 설정되어 있어야 합니다:

| 항목 | 설정 방법 |
|------|-----------|
| STT (텍스트 변환) | 설정 → STT: 사용 |
| AI 요약 + API 키 | 설정 → AI 요약: 사용, API 키 입력 |
| 텔레그램 + 봇 토큰 + Chat ID | 설정 → 텔레그램: 사용, 토큰/ID 입력 |

미충족 시 안내 메시지가 나오고 설정 화면으로 이동할 수 있습니다.

## 문제 해결 가이드

에러가 나면 이 섹션에서 찾아보세요.

### Docker 관련 문제

#### "docker" 명령어를 찾을 수 없다고 나와요

```
'docker'은(는) 내부 또는 외부 명령이 아닙니다.
또는
command not found: docker
```

**원인**: Docker Desktop이 설치되지 않았거나 실행 중이 아닙니다.

**해결**:
1. Docker Desktop이 설치되어 있는지 확인 → 없으면 [1단계](#1단계-docker-desktop-설치)로
2. 설치되어 있다면 Docker Desktop을 실행하세요
3. 작업표시줄(Windows) 또는 메뉴바(Mac)에 고래 아이콘이 보이고, Docker Desktop 왼쪽 하단에 **초록색 Engine running**이 뜰 때까지 기다리세요
4. 다시 시도

#### WSL 관련 오류가 나와요 (Windows)

```
WSL 2 installation is incomplete
또는
Please enable the Virtual Machine Platform
```

**해결**:
1. 시작 메뉴에서 "PowerShell" 검색 → **우클릭** → **관리자 권한으로 실행**
2. 아래 명령어 입력:

```powershell
wsl --install
```

3. **컴퓨터 재시작**
4. Docker Desktop 다시 실행

#### "no matching manifest" 오류

```
no matching manifest for windows/amd64 in the manifest list
```

**원인**: Docker가 Linux 컨테이너 모드가 아닙니다.

**해결**:
1. 작업표시줄의 Docker 고래 아이콘을 **우클릭**
2. **"Switch to Linux containers..."** 클릭
3. 다시 시도

#### "Cannot connect to the Docker daemon" 오류

```
Cannot connect to the Docker daemon at unix:///var/run/docker.sock
```

**원인**: Docker Desktop이 아직 완전히 시작되지 않았습니다.

**해결**: Docker Desktop을 열고 **30초~1분** 기다린 후 다시 시도하세요.

#### 이미지 다운로드(pull)가 너무 느려요

**원인**: 네트워크 환경 또는 Docker Hub 서버 상태에 따라 느릴 수 있습니다.

**해결**: 기다려주세요. 최초 1회만 오래 걸리고, 다음부터는 빠르게 시작됩니다.

### 로그인 관련 문제

#### "로그인에 실패하였습니다"

**확인할 것**:
1. 학번과 비밀번호가 맞는지 확인
   - 웹 브라우저에서 https://canvas.ssu.ac.kr 에 직접 로그인 시도
   - 같은 학번/비밀번호로 로그인되면 프로그램 문제
   - 로그인 안 되면 비밀번호 문제 → LMS에서 비밀번호 재설정
2. 비밀번호에 특수문자(`@`, `!`, `#` 등)가 있어도 **그대로 입력**하세요
3. 학번 앞에 공백이나 뒤에 공백이 없는지 확인

#### 로그인은 성공했다고 나오는데 과목 목록이 안 떠요

**원인**: LMS 서버가 느리거나 일시적 장애일 수 있습니다.

**해결**:
1. 프로그램을 종료(`Ctrl+C`)하고 다시 실행
2. 그래도 안 되면 웹 브라우저에서 LMS 사이트가 정상인지 확인
3. LMS 점검 시간(보통 새벽)에는 동작하지 않을 수 있습니다

#### 비밀번호를 변경했더니 로그인이 안 돼요

**원인**: 이전 비밀번호가 암호화되어 `.env`에 저장되어 있습니다.

**해결**: `.env` 파일을 열어서 `LMS_PASSWORD=` 줄의 값을 지우고 다시 실행하세요.

```
# 변경 전
LMS_PASSWORD=enc:gAAAA...

# 변경 후 (= 뒤를 비우면 됨)
LMS_PASSWORD=
```

다시 실행하면 비밀번호를 새로 입력하라고 나옵니다.

### 재생 관련 문제

#### 텔레그램으로 "[오류] ... 재생을 실패하였습니다" 알림이 와요

**원인**: LMS 서버 상태 또는 네트워크 문제로 재생에 실패했습니다.

**해결**:
1. **자동 모드**에서는 최대 3회 자동 재시도하므로, 1~2번 실패해도 대부분 다음 시도에서 성공합니다
2. 계속 같은 강의가 실패하면:
   - 웹 브라우저에서 해당 강의를 직접 재생해보세요
   - 강의 자체가 깨져 있으면 교수님이나 LMS 관리자에게 문의
3. 모든 강의가 실패하면:
   - LMS 사이트(canvas.ssu.ac.kr)가 정상인지 확인
   - 프로그램을 재시작

#### 재생 완료라고 나왔는데 LMS에서 출석이 안 잡혀요

**확인할 것**:
1. LMS에서 새로고침(F5) 한번 눌러보세요 — 반영이 느릴 수 있습니다
2. 그래도 안 되면 해당 강의를 다시 재생해보세요
3. 일부 강의는 "learningx" 형식이라 출석 방식이 다릅니다 — 프로그램이 자동으로 처리하지만, 드물게 누락될 수 있습니다

### 다운로드 관련 문제

#### 텔레그램으로 "[오류] ... 다운로드에 실패하였습니다" 알림이 와요

**원인** (여러 가지 가능):
1. 영상 URL 추출 실패 — LMS에서 영상 주소를 못 찾은 경우
2. CDN 서버 오류 — 영상 파일 서버가 일시적으로 응답하지 않는 경우
3. 네트워크 끊김 — 다운로드 중 인터넷이 불안정한 경우

**해결**:
1. 프로그램을 다시 실행해서 해당 강의를 다시 다운로드
2. 자동 모드에서는 다음 스케줄에서 자동 재시도합니다
3. 특정 강의만 계속 실패하면:
   - `logs/` 폴더에서 최신 로그 파일 확인 (에러 원인이 적혀 있음)
   - 해당 강의가 "다운로드 불가" 형식(learningx)일 수 있음

#### "다운로드 불가: 이 강의는 다운로드가 지원되지 않는 형식입니다"

**원인**: 일부 강의는 "learningx" 형식으로 되어 있어 구조적으로 영상 파일을 추출할 수 없습니다.

**해결**: 이 경우 다운로드는 불가하지만, **재생(출석 처리)은 정상 동작**합니다.

#### 다운로드된 파일은 어디 있나요?

`study-helper` 폴더 안의 `data/downloads/` 경로:

```
study-helper/
└── data/
    └── downloads/
        └── 과목명/
            └── 1주차/
                ├── 강의명.mp4    ← 영상 파일
                ├── 강의명.mp3    ← 음성 파일
                ├── 강의명.txt    ← STT 텍스트
                └── 강의명_summarized.txt  ← AI 요약
```

### 텔레그램 관련 문제

#### 텔레그램 알림이 안 와요

**확인할 것**:
1. 설정에서 텔레그램이 **사용**으로 되어 있는지 확인
2. 봇 토큰이 맞는지 확인 — BotFather에서 봇을 찾아 토큰 재확인
3. Chat ID가 맞는지 확인 — @userinfobot에서 다시 확인
4. **중요**: 봇에게 **먼저 아무 메시지를 보내야** 봇이 알림을 보낼 수 있습니다
   - 텔레그램에서 봇을 찾아 `/start`를 보내세요

#### "메시지 전송 실패. Chat ID를 확인하세요"

**원인**: Chat ID가 잘못되었거나 봇에게 메시지를 보내지 않았습니다.

**해결**:
1. 텔레그램에서 @userinfobot을 검색 → `/start` → Id 확인
2. 봇에게 `/start` 메시지를 보냈는지 확인
3. 설정에서 Chat ID를 다시 입력

#### 봇 토큰을 분실했어요

1. 텔레그램에서 @BotFather 검색
2. `/mybots` 입력
3. 해당 봇 선택 → API Token 확인

### AI 요약 관련 문제

#### "AI 요약 실패"

**원인**:
1. API 키가 잘못되었거나 만료됨
2. Gemini API 무료 한도 초과 (분당 요청 수 제한)
3. 네트워크 문제

**해결**:
1. [Google AI Studio](https://aistudio.google.com)에서 API 키가 유효한지 확인 (Gemini API 키 = Google API 키, 같은 것입니다)
2. 무료 한도는 보통 분당 15건 — 잠시 기다렸다가 다시 시도
3. 키를 새로 발급받아서 설정에 다시 입력

#### 요약이 너무 짧거나 이상해요

**해결**: 설정에서 **추가 요약 지시사항**에 원하는 내용을 입력하세요.

예시:
- `한국어로 상세하게 요약해주세요`
- `핵심 개념 위주로 정리하고 시험에 나올 만한 부분을 강조해주세요`
- `목차 형식으로 정리해주세요`

### STT(음성→텍스트) 관련 문제

#### STT 변환이 너무 오래 걸려요

**원인**: 강의 길이와 Whisper 모델 크기에 따라 시간이 달라집니다.

| 모델 | 30분 강의 기준 예상 시간 | 정확도 |
|------|------------------------|--------|
| tiny | 1~2분 | 낮음 |
| base | 3~5분 (기본값) | 보통 |
| small | 5~10분 | 좋음 |
| medium | 10~20분 | 높음 |
| large | 20분 이상 | 최고 |

**해결**: 시간이 너무 오래 걸리면 설정에서 모델을 `tiny`로 변경하세요. 정확도가 떨어지지만 빠릅니다.

#### STT 결과가 엉망이에요

**원인**: 강의 음질이 나쁘거나 모델이 너무 작습니다.

**해결**:
1. 모델을 `small` 또는 `medium`으로 변경 (더 정확하지만 느림)
2. 언어 설정이 `ko`(한국어)로 되어 있는지 확인
3. 영어 강의라면 언어를 `en`으로 변경

### 자동 모드 관련 문제

#### "자동 모드 실행을 위한 필수 조건이 만족하지 않았습니다"

아래 3가지를 **모두** 설정해야 합니다:

| 확인 항목 | 설정 방법 |
|----------|-----------|
| STT 활성화 | setting → STT: `사용` 선택 |
| AI 요약 + API 키 | setting → AI 요약: `사용` + Gemini API 키 입력 |
| 텔레그램 + 봇 + Chat ID | setting → 텔레그램: `사용` + 봇 토큰 + Chat ID 입력 |

`y`를 입력하면 설정 화면으로 바로 이동합니다.

#### 자동 모드를 켜놨는데 아무것도 안 해요

**확인할 것**:
1. "다음 체크 HH:MM (N시간 후)" 메시지가 표시되고 있나요?
   - 표시되고 있다면 → 다음 스케줄 시간까지 대기 중입니다. 정상입니다.
   - `즉시 실행` 옵션을 `y`로 선택하면 바로 시작합니다
2. "미시청 강의가 없습니다" 메시지가 나왔나요?
   - LMS에서 모든 강의를 이미 시청한 상태입니다

#### 자동 모드 중 컴퓨터를 꺼도 되나요?

**안 됩니다.** 컴퓨터가 꺼지거나 Docker가 종료되면 자동 모드도 중단됩니다.

자동 모드를 사용하려면:
- 컴퓨터를 켜둔 상태로 유지
- Docker Desktop이 실행 중이어야 함
- 터미널(PowerShell/터미널) 창을 닫지 말 것

> 노트북 사용자: 덮개를 닫으면 절전 모드로 들어가 자동 모드가 중단될 수 있습니다. 절전 설정을 변경하세요.

### 기타 문제

#### 프로그램이 멈추거나 응답이 없어요

**해결**:
1. `Ctrl + C`를 눌러보세요
2. 한 번에 안 되면 `Ctrl + C`를 **두 번** 누르세요
3. 그래도 안 되면 터미널 창을 닫고 다시 실행

#### 프로그램을 업데이트하고 싶어요

```bash
docker compose pull
docker compose run --rm study-helper
```

이 두 줄만 실행하면 최신 버전으로 업데이트됩니다.

#### 모든 설정을 초기화하고 싶어요

`study-helper` 폴더에서 아래 파일을 **삭제**하면 초기화됩니다:

| 파일 | 삭제 시 효과 |
|------|-------------|
| `.env` | 모든 설정 초기화 (학번, 비밀번호, API 키 등) |
| `.secret_key` | 암호화 키 초기화 (저장된 비밀번호 복호화 불가) |

> 두 파일 모두 삭제 후 다시 실행하면 처음부터 설정을 시작합니다.

삭제 후 `.env` 파일을 다시 받으세요:

```bash
curl -o .env https://raw.githubusercontent.com/igor0670/study-helper/main/.env.example
```

#### 디스크 용량이 부족해요

Docker 이미지와 다운로드 파일이 용량을 차지합니다.

**해결**:
1. `data/downloads/` 폴더에서 불필요한 파일 삭제
2. Docker 캐시 정리:

```bash
docker system prune -a
```

> 위 명령어는 사용하지 않는 Docker 이미지를 모두 삭제합니다. 다음 실행 시 이미지를 다시 다운로드합니다.

#### `logs/` 폴더의 로그 파일은 뭔가요?

에러 발생 시 자동으로 로그 파일이 생성됩니다.

```
logs/
├── study_helper.log           ← 전체 실행 로그
├── 20260322_180123_play.log   ← 재생 에러 상세 로그
└── ...
```

문제를 해결할 수 없을 때 이 파일을 개발자에게 보내면 원인 파악에 도움이 됩니다.
